Bookstore Web App

Это мой финальный проект курса «CS50’s Web Programming with Python and JavaScript». Ниже описаны все необходимые детали согласно требованиям этого проекта.

Distinctiveness and Complexity

Данный проект позиционируется как книжный интернет-магазин. Он отличается от проекта 2 "e-commerce" текущего курса по таким ключевым пунктам:

  1. Архитектура Базы Данных. В Bookstore по-другому представлена архитектура Базы Данных, данные пользователя хранятся совершенно иначе.
  2. Принцип работы. Товары в Bookstore не являются уникальными в своём единичном представлении, как это было в Проекте 2 с лотами. Это позволяет пользователю многократно приобретать товары во многих экземплярах.
  3. Дизайн. Несмотря на использование библиотеки Bootstrap в обоих проектах, в Bookstore совершенно другое дизайнерское решение, соответствующее общим представлениям об интернет-магазине, а не интернет-аукционе.
Далее описаны пункты, определяющие сложность, уникальность и удобство веб-приложения.

Архитектура БД и управление данными

Принцип хранения и управления данными выполнен следующим образом:

  • Неавторизованный пользователь может смотреть список книг, а также посмотреть страницу отдельной книги. Для добавления книг в корзину, а также для просмотра своей корзины пользователю необходимо авторизоваться.
  • Таблица Purchase хранит информацию о покупках (корзинах) пользователей. Ключевым является поле paid, которое определяет, оплачена ли корзина. Как только корзина становится оплаченной, поле purchase_date автоматически заполняется текущей датой. Логика покупок пользователя продумана так, что у одного пользователя может быть активна только одна корзина (у которой paid=False).
  • Таблица Purchase Item хранит записи о каждой книге, которая находится в корзине пользователя (вне зависимости от того, оплачена корзина или нет), в каком количестве и на какую сумму вышла данная книга (с учётом количества).
  • Общая сумма всей корзины хранится в ранее упомянутой таблице Purchase. В логике работы с данными предусмотрен пересчёт этого значения каждый раз, когда пользователь изменяет какие-либо данные в своей корзине (количество книг или вообще их наличие).

Роль JavaScript

В Bookstore используется JavaScript для разных задач и страниц, чтобы сделать опыт пользования сайта достаточно комфортным. Вот несколько примеров того, в каких местах и с какой целью это используется:

  • На главной странице над книгами находится панель с поиском по книгам и фильтрами по цене/категории. Код полностью предусматривает все возможные комбинации фильтров с поисковой строкой.
  • На странице конкретной книги находится блок по добавлению товара в корзину. JS-код предусматривает:
    • Правильную работу поля Amount. Поле является текстовым и защищено от ввода любых символов, кроме чисел (вместо не числового символа в поле записывается "1"). Если товар есть в корзине, то поле ввода и текст возле "Amount" содержат значение количества данной книги в корзине. Если книг в корзине нет, то в поле ввода содержится "1"
    • Удобное изменение количества книг с помощью кнопок + и -. Они успешно добавляют/отнимают количество книг по одной за нажатие, а также блокируются, если в поле записано максимально или минимально возможное число соответственно.
    • Добавление товара в корзину при нажатии на кнопку Add to Cart. Она будет заблокирована, если поле количества книг пустое.
    • Следующую логику работы кнопок: если товар находится в корзине (не важно, добавил ли пользователь только что или раньше), вместо кнопки Add to Cart появляются кнопки Change amount и Remove from Cart. Кнопка Change amount изменяет количество книг в корзине на новое значение поля Amount и блокируется по принципу кнопки Add to Cart, а Remove from Cart удаляет товар из корзины, после чего эти две кнопки удаляются и появляется снова Add to Cart
  • На странице корзины пользователя есть рядом с названием каждого товара для удобного удаления из корзины без посещения страницы книги. Если все товары из корзины будут удалены через кнопку удаления или с помощью кнопки Purchase (имитирует процесс покупки), появится шаблон пустой корзины.
  • На странице обзора предыдущих покупок пользователю показывается его самая популярная категория из всех купленных им книг с указанием количества.

Роль HTML/CSS

Конечно, HTML является обязательным для использования на проекте, но по возможности используются подходящие семантические теги в нужных местах. Но основное, что хочется отметить во внешнем виде, это использование CSS:

  • Практически повсеместно используется Flexbox разметка и классы Bootstrap. Сайт хорошо адаптирован к разным экранам.
  • На странице конкретной книги главной раскладкой является Grid для удобного кастомного позиционирования блоков при изменении ширины экрана

Запуск веб-приложения

Чтобы запустить и протестировать работу Bookstore, выполните следующие шаги:

  1. Убедитесь, что у вас установлен Python версии 3.11+ и Django 4.1+.
  2. Скачайте проект на компьютер.
  3. Откройте терминал вашей ОС.
  4. Перейдите в главный каталог проекта, где расположен скрипт manage.py
  5. Запустите manage.py с параметром runserver. Вот как это может выглядеть: py manage.py runserver
  6. Перейдите на сайт проекта по ссылке (по умолчанию: 127.0.0.1:8000)
